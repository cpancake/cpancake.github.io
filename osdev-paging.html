Remember how I said we're going to be talking about the GDT this time around? Well, I was going to. But then I came across this article, and I changed my mind. Today, we're going to be talking about paging.

What is paging?

Paging is the modern way of handling memory. Back in the old system - the GDT - it used a system called segmentation. Basically, you'd tell the computer that some part of memory was the kernel's data, another part was the kernel's code, another part was user code, and another part was user data. That way, the user code couldn't conflict with the operating system. This works all fine and dandy, but what if you need to run multiple processes? With only the GDT, multiple processes have the same memory space. If two programs both write to 0x300, one program will get the wrong data. This was previously solved with the LDT, but the modern way of doing it is with paging.

Paging is assigning different "pages" - small sections of memory - to physical memory. When you run a program, you assign pages to that program.
